# 開発順序

## 基本方針

- 最小限動くものを先に作り、動かしながら調整する
- パラメータとシミュレーションは相互依存するので、仮の値で進めて後で調整
- 画面は必要になったタイミングで作る

---

## Phase 1a: 野球ルールエンジン（能力一律） ✅ 完了

**目標**: 野球のルールに従って1試合シミュレーションし、スコアボードで結果を見れる

### 単純化ルール（Phase 1a）
- 投手交代なし（1人が9回投げ切る）
- 代打なし
- 延長なし（9回で同点でも終了）
- 併殺・タッチアップ・盗塁なし
- エラーなし

### Step 1a-1: 1球のシミュレーション ✅
- [x] 投球結果をランダムで決定
- [x] ボール/ストライク/ファウル/インプレー
- [x] カウント管理（B-S）
- [x] 球速パラメータ（毎球±5kmの変動）

### Step 1a-2: 1打席のシミュレーション ✅
- [x] 投球ループ（打席決着まで）
- [x] 三振/四球/インプレーの判定
- [x] 打球結果（アウト/単打/二塁打/三塁打/本塁打）

## Phase 1b: 能力パラメータ導入

**目標**: 選手の能力によって結果が変わるようにする

### Step 1b-1: 選手パラメータ定義
- 野手: ミート、長打力（最小構成）
- 投手: 制球、奪三振力（最小構成）

### Step 1b-2: 確率計算に能力を反映
- 投手 vs 打者の能力差で確率補正
- 例: ミート高い → ヒット確率UP

### Step 1b-3: バランス調整
- 100試合ほど回して成績分布確認
- 打率/本塁打/防御率がリアルな範囲に収まるよう調整

### Step 1b-4: パラメータ拡張
- 走力、守備力、スタミナ等を追加
- 段階的に影響を実装

**成果物**: 能力に応じた結果が出る試合シミュレーション

## まとめ

**最初にやること**: Step 1-1〜1-5（1試合のシミュレーション）

パラメータは仮の値で始めて、実際に試合を回しながら「ホームランが出すぎる」「三振が多すぎる」等を調整していく。

動くものを先に作ることで、モチベーション維持にもなる。
