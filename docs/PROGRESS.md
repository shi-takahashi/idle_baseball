### パラメータ
- [x] 投手の球速パラメータ（平均球速、±5kmの変動）
- [x] 球速による結果確率の補正
- [x] 投手の制球力パラメータ（1〜10段階）
  - 制球力が高いほどボール球が減少（25%〜45%の範囲）
  - 制球力が高いほど甘い球が減り、被打率が低下
- [x] 野手のミート力パラメータ（1〜10段階）
  - ミート力が高いほど空振りが減少 → インプレー（打球が飛ぶ）確率が上昇
  - インプレー後のヒット/アウト判定には影響しない（バットに当てる力）
- [x] 野手の長打力パラメータ（1〜10段階）
  - 長打力が高いほどホームラン確率が大幅に上昇
  - 長打力が高いほど二塁打・三塁打確率も上昇
  - 長打力が高いほど単打もわずかに増加（打球速度）
- [x] 打球方向の概念追加
  - FieldPosition列挙型（投手/捕手/一塁/二塁/三塁/遊撃/左翼/中堅/右翼）
  - 打球の種類（ゴロ/フライ/ライナー）によって飛ぶ方向の確率が変化
  - ゴロは内野、フライは外野メイン、ライナーは全方向に分散
- [x] 守備力パラメータ（ポジションごと、0〜10）
  - DefensePosition列挙型（捕手/一塁/二塁/三塁/遊撃/外野）
  - 外野は左翼・中堅・右翼をまとめて1つ
  - 0はそのポジションを守れない、1〜10は守備力
  - 守備力が高いほどヒットになりにくい（アウト率が上がる）
  - チームに守備配置（誰がどこを守るか）を設定可能
- [x] 走力パラメータ（1〜10）
  - 盗塁試行確率と盗塁成功率に影響
- [x] 盗塁の概念追加
  - 盗塁可能な状況を判定（ランナーの位置による）
  - 1塁のみ→2塁へ、2塁のみ→3塁へ、1,2塁→ダブルスチール
  - **1球ごとに盗塁判定**（打席中いつでも盗塁可能）
  - 盗塁成功/失敗は走力に基づいて判定
  - ダブルスチール: 2塁ランナーの走力で成否判定、失敗時は2塁ランナーのみアウト
  - 投球結果との組み合わせ処理:
    - 四球時、押し出し対象のランナーは盗塁記録なし
    - 三振+盗塁成功/失敗も正しく処理
  - ホームスチールは未実装
- [x] イニング詳細で盗塁表示
  - 投球チップの横に盗塁成功/失敗を表示

### 2026-04-22 併殺打（ダブルプレー）の実装

**実装内容:**
- ランナー1塁+アウト<2でゴロアウト時に併殺判定を行う
- `AtBatResultType.doublePlay`を追加し、結果画面で「併殺打」と表示
- 走力に応じて「併殺」または「併殺崩れ（通常ゴロアウト）」が決まる

**併殺成功率（打者の走力）:**
- 走力1: 94%（ほぼ必ず併殺）
- 走力5: 70%（基準）
- 走力10: 40%（併殺崩れが多い）

**併殺打時の処理:**
- 1塁ランナー: 2塁でフォースアウト（消える）
- 打者: 1塁でアウト
- 2塁ランナー: 3塁に進む
- 3塁ランナー: 基本動かない、満塁時はホームへ（得点）
- ただし併殺で3アウトになる場合は得点なし
- アウトカウント: +2

**テスト結果（10試合×2チーム）:**
- 鈍足チーム（走力1）: 併殺打12回
- 俊足チーム（走力10）: 併殺打2回

**今後の拡張可能性:**
- ランナー1,2塁での併殺（6-4-3など）
- ライナーゲッツー
- ホームゲッツー

### 2026-04-22 内野安打の実装

**実装内容:**
- ゴロアウトの一部が、打者の走力次第で内野安打になる
- `AtBatResultType.infieldHit`を追加し、単打と区別して表示
- 結果画面では「内野安打」「内安」と表示される
- 確率計算: `走力 × 1.2% × 方向補正 × 守備力補正`

**打球方向による補正:**
- サード/ショート: ×1.5（一塁から遠い）
- セカンド: ×1.0
- ファースト: ×0.3（一塁に近い）
- 投手/捕手: ×0.5

**テスト結果（10試合×2チーム）:**
- 鈍足チーム（走力1）: 単打45、内野安打2、ゴロアウト82
- 俊足チーム（走力10）: 単打49、内野安打10、ゴロアウト71

### 2026-04-22 走力による追加進塁

**実装内容:**
- シングル/ツーベースで「+1塁」の追加進塁が走力依存で発生
- 確率: `走力 × 5%`（走力1で5%、走力10で50%）
- 前のランナーが詰まっている場合は追加進塁不可

**具体例:**
- 2塁ランナー + シングル → 通常3塁、走力次第でホーム
- 1塁ランナー + ツーベース → 通常3塁、走力次第でホーム
- 1,2塁でシングル → 2塁ランナーがホームなら1塁ランナーは3塁可、そうでなければ2塁止まり

**テスト結果（10試合×2チーム）:**
- 鈍足チーム（走力1）: 安打103本で43得点（0.42点/安打）
- 俊足チーム（走力10）: 安打94本で56得点（0.60点/安打）

### 2026-04-22 盗塁システム改修

**大幅なアーキテクチャ変更:**
- 盗塁判定を「打席前に1回」から「1球ごと」に変更
- `AtBatSimulator.simulateAtBat`が`BaseRunners`、`outs`、`StealSimulator`を受け取るように
- 戻り値を`AtBatSimulationResult`クラスに変更（盗塁情報含む）

**ダブルスチールのルール:**
- キャッチャーは3塁に送球（短い距離を優先）
- 2塁ランナーの走力で成否判定
- 成功: 両者とも盗塁成功として記録
- 失敗: 2塁ランナーはアウト、1塁ランナーは2塁に進塁（盗塁記録なし）

**投球+盗塁の組み合わせ:**
- `StealAttempt`に`isOut`フィールド追加（アウトかどうかを区別）
- 四球時、押し出し対象のランナーは盗塁記録なし
- 盗塁失敗+ストライク3 = 2アウト（三振+盗塁死）

**次回の予定:**
- 走力による進塁判定（単打で1塁から3塁など）
- その他走力関連の機能

### 2026-04-22 タッチアップ（犠牲フライ）の実装

**実装内容:**
- 外野フライ時、2アウト未満であればランナーがタッチアップを試行
- 走力に応じて試行確率と成功確率が変化
- 打球方向（FieldPosition）を`_advanceRunners`に渡して判定

**3塁ランナーのタッチアップ:**
- 全ての外野フライで試行可能
- 試行確率: `速度 × 7.8% + 2%`（10%〜80%）
- 成功確率: `速度 × 6% + 34%`（40%〜95%）
- **失敗時はアウト**（ランナーは消える）

**2塁ランナーのタッチアップ:**
- ライト/センターフライのみ試行可能（レフトは3塁に近すぎる）
- 3塁ランナーがいる場合: バックホームなので3塁ランナーの成否に連動
- 3塁ランナーがいない場合: 自身の走力で判定
- **失敗時はアウト**（ランナーは消える）

**タッチアップ失敗によるアウト:**
- `_RunnerAdvanceResult`に`additionalOuts`フィールドを追加
- タッチアップ失敗で3アウトになった場合、その打席での得点は無効

**テスト結果（50試合×2チーム）:**
- 鈍足チーム（走力1）: 犠牲フライ2回（フライアウト368回、成功率0.5%）
- 俊足チーム（走力10）: 犠牲フライ19回（フライアウト371回、成功率5.1%）

**これで走力パラメータ関連の実装が完了:**

- 併殺回避
- タッチアップ（犠牲フライ）
