### 2026-05-06 先発投手の交代判断の総合化（完了）

**動機:**
ユーザー指摘 — 初回に2点取られた時点で先発が降板するなど、交代判断が
早すぎた。原因は「3連打」が単独で降板トリガーになっていたこと、および
失点ラインがイニング非依存（一律5失点）で QS の目安を反映していなかったこと。
リリーフ陣の負担が過大になっていた。

**変更:**
- `SimplePitcherChangeStrategy._decideStarterPull` を新設（先発専用の判断ロジック）
- 抑え・リリーフは従来ロジックのまま（イニング境界の25球スイッチや
  5失点・3連打・3連続四球の即時降板は維持）
- 先発の判断を以下の総合判断に変更:
  1. 球数到達（100球）
  2. **イニング相応の失点超過**: 1〜3回 6失点 / 4〜5回 5失点 / 6回以降 4失点（QS 崩壊）
  3. 4連打 / 4連続四球（深刻な制球乱れ）
  4. 3連打 / 3連続四球は **(60球以上 OR 4失点以上 OR 5回以降)** と組み合わせて判定
  5. 終盤のピンチ（既存。7回以降 + 同点以下 + 得点圏 + 50球以上）

**検証 (`bin/measure_starter.dart`、3シーズン × 6チーム):**

| 指標            | 修正前        | 修正後        |
|-----------------|---------------|---------------|
| 平均IP          | 5.67          | 6.01          |
| 初回降板率      | 3.9% (21件)   | 0.9% (5件)    |
| 3回までの早期降板 | 11.3% (61件) | 4.3% (23件)   |
| `<3IP` 比率     | 11.5%         | 4.3%          |
| 「3連打」降板   | 76件          | 36件          |

NPB 平均（先発IP 5.5〜6.0、初回降板 1% 未満）に近づいた。

### 2026-05-06 盗塁発動カーブの非線形化（完了）

**動機:**
ユーザー指摘 — 走力 5（平均）の走者がダブルスチールを仕掛けるなど違和感があった。
「盗塁は走力の高い選手の特権」というデザインに合わせ、平均走力以下は仕掛けず、
走力 7〜8 から急に増えるカーブに変更。試行した場合の成功率は基本高め。

**変更:**
- 試行確率を線形補正 → **走力ごとの非線形テーブル**に変更:

  | 走力 | 1球あたり試行確率 |
  |------|-------------------|
  | 1    | 0 (仕掛けない、走力差を明確化) |
  | 2    | 0.05%             |
  | 3    | 0.12%             |
  | 4    | 0.25%             |
  | 5    | 0.40%             |
  | 6    | 0.9%              |
  | 7    | 2.2% ← 急増の閾値 |
  | 8    | 4.2%              |
  | 9    | 6.5%              |
  | 10   | 8.5%              |

- 成功率の基本値を 70% → **75%** に引き上げ（仕掛けた以上は基本高め）
- 走力ボーナス: +5% → **+3%** / 1点（基本値が高いので係数を抑える）
- 捕手肩ペナルティ: -2.5% / 1点（変更なし）
- ダブルスチールは 2 塁走者の走力で起動判定（変更なし。3 塁送球が短いので
  3 塁を奪いに行く走者の脚が決め手）

**検証 (`bin/measure_steals.dart`、3シーズン × 6チーム = 540 team-games):**

| 1チーム1試合あたり成功     | 0.60          | 0.39          |

起動者の 86%（198 / 231）が走力 6 以上、64%（147 / 231）が走力 7 以上で、

### 2026-05-06 外野ライナーのアウト率修正（完了）

**動機:**
ユーザー指摘 — NPB 実測では外野ライナーのアウト率は 27〜30%（残り約7割が安打）、
外野フライは 67〜72%（多くがアウト）。逆転していたら NG という基準。

**原因:**
`_calculateInPlayProbabilities` が `battedBallType` を考慮せず、ライナー・フライ・
ゴロが**同じ probOut（基本 0.70）** で判定されていた。打球種別はアウト結果型
（lineOut / flyOut / groundOut）のラベルにしか使われていなかった。

**変更:**
- `_calculateInPlayProbabilities` に `battedBallType` と `fieldPosition` を渡す
- 打球タイプ × 外野/内野方向で probOut を補正:
  - 外野ライナー: -0.58（強い打球が外野手の前後左右に落ちやすい）
  - 内野ライナー: 補正なし（反応速度で多くがアウト = lineOut）
  - 外野フライ: 補正なし（既存値で 70% 付近に合致）
  - 内野フライ（ポップフライ）: +0.10（ほぼ確実にアウト）
  - ゴロ: 補正なし

**検証 (`bin/measure_outfield_balls.dart`、5 シーズン × 6 チーム):**

| 打球タイプ        | 修正前    | 修正後    | 現実値      |
|-------------------|-----------|-----------|-------------|
| 外野ライナー      | 71.0%     | 28.9%     | 27〜30%     |
| 外野フライ        | 69.7%     | 68.5%     | 67〜72%     |
| ゴロ（参考）      | 65.3%     | 65.2%     | 65〜75%     |

外野ライナー > 外野フライの逆転を解消し、NPB 実測のど真ん中に着地。

## 短いメモ

- 打者一巡時はカンマ区切りで表示（例: `安打, 三振`）
- 2アウトでゴロアウト時は得点が入らないよう修正済み
- 球速は毎球±5kmの変動（中央寄りの分布）
- 野手の調子は隠しパラメータ（UIには出さない）。RecentForm（成績ベース）と役割分担済み
- 球速の編集範囲は 100〜165 km/h（試合内の調子で +5 km/h 揺れて 170 km/h ≒ メジャー最高記録）
- 自チームの SP 自動選出: `_pickFreshestStarter`（コンディション高い順 → 背番号順）
- 他チームの SP: `_selectStarter`（中4日 + エースバイアス + ジッター、リーグ多様性のため別ロジック）
- 後でやりたい: 背番号もチーム編成画面で設定できるようにする
- 後でやりたい: 試合開始ボタンを「real-time の時刻判定」と組み合わせて、時間が来てたら直接結果へ
